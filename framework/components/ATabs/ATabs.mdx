# Tabs

## Install

Integrate with your build to [auto-import](/#integrating), or add an import in your component:

```
import {ATabGroup, ATab} from "@cisco-sbg-ui/magna-react";
```

## Usage

#### \*\*\*\*Note regarding Primary v. Secondary - There is no use case for Primary currently so we are setting Secondary to true.

#### If you need primary immedaitely, set secondary to false and please reach out to magna team so we can coordinate with UX on the path forward to implimenting primary as default.

#### \*\*\*\*

### Uncontrolled Mode

<Playground
  code={`<>
  <div>

<div className="mt-5">

### Controlled Mode

<Playground
  code={`() => {
  // Add a comment
  const tabs = ["One", "Two", "Three", "Four", "Five", "Six", "Seven"];
  const [activeTab, setActiveTab] = useState(tabs[2]);
  return (
    <ATabGroup>
      {tabs.map((x) => (
        <ATab
          key={x}
          tabKey={x}
          onClick={() => setActiveTab(x)}
          selected={x === activeTab}>
          {x}
        </ATab>
      ))}
    </ATabGroup>
  );
}
`}
/>

## Router Integrations

On `ATab`, set the `component` property to your router's link type, for example:

```
// With react-router:
<ATab component={RouterLink} to="/">Tab 1</ATab>
```

## Accessibility Notes

By default, `ATabGroup` components are assigned the [WAI-ARIA](https://www.w3.org/WAI/standards-guidelines/aria/) role of [tablist](https://www.w3.org/TR/wai-aria/#tablist).

By default, `ATab` components are assigned the [WAI-ARIA](https://www.w3.org/WAI/standards-guidelines/aria/) role of [tab](https://www.w3.org/TR/wai-aria/#tab).

The `ATab` `click` event handler is triggered both by the `click` event and by the `keyDown` event for the `Enter` key.

Each `ATab` component is tab-focusable with a visual overlay.

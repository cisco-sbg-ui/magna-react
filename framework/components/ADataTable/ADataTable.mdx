##### Disable Column Sort Resets

By default, when you click a sortable column for a third time, it will reset the sorts, i.e., the column will return to an unsorted state.

However, if you prefer for the column to mainting the sorting state after subsequent clicks, pass `disableSortReset={true}`to `ADataTable`.

> Please note that with this option enabled, the table will **stay sorted** (either ascending or descending) after the user initiates a sort.

#### Seletable Rows

To make row items selectable, pass both the `onRowClick` and `isRowSelected` props to `ADataTable`. Both of these props expect a function that will be called with the row item's data so that you can return a truthy or falsy value to indicate the selected state.

> The example below only handles one row being selected at a time. To handle multiple rows being selected at once, you'll want to update the way you store selected rows in state and perform the proper conditional for `isRowSelected`.

<Playground
  code={`function Example() {
  const [selectedWorld, setSelectedWorld] = useState();
  const headers = [
    {
      name: "World",
      key: "world",
    },
    {
      name: "Players",
      key: "players",
    },
    {
      name: "Location",
      key: "location",
    },
  ];
  const items = [
    {
      world: "World 1",
      players: 1238,
      location: "United States",
    },
    {
      world: "World 34",
      players: 189,
      location: "United States",
    },
    {
      world: "World 81",
      players: 72,
      location: "United Kingdom",
    },
  ];
  return (

      isRowSelected={(rowItem) => rowItem.world === selectedWorld}

  );
}
`}

#### With a Checkbox

<Playground

## Accessibility Notes

When using infinite scroll within the `ADataTable`, it is recommended to set a maxinumum infinite scrolling threshold in your implementation so that screen readers and keyboard users can also have a good user experience. To continue loading more content after reaching the target threshold, you can use the "Load More" pattern by rendering a button that will then begin to explicitly load more table content with the user's consent.

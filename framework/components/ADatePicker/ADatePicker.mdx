#### Disabling Dates

To disable certain dates from being selectable, you can pass a `minDate`, `maxDate`, or both. Suppling only one will act as a floor or ceiling respectively.

In the example below, the user is only permitted to select a date within a week from the current date.

> As of the latest Magna React release, it is only possible to disable consecutive days.

<Playground
  code={`function Example() {
  const today = new Date();
  const weekFromToday = new Date(today);
  weekFromToday.setDate(today.getDate() + 7);
  const [date, setDate] = useState(today);
  return (
    <ADatePicker
      value={date}
      onChange={(date) => setDate(date)}
      minDate={today}
      maxDate={weekFromToday}
    />
  );
}
`}
/>

#### Date Ranges

To render a date range, you can import the `useADateRange` hook, which takes care of the logic for you:

<Playground
  code={`function Example() {
  const today = new Date();
  const weekFromToday = new Date(today);
  weekFromToday.setDate(today.getDate() + 7);\n
  const { value, onChange, } = useADateRange({
    initialRange: [today, weekFromToday]
  });\n
  return (
    <ADatePicker value={value} onChange={onChange} />
  );
}
`}
/>

The logic for `useADateRange` ensures that the first and second clicks will alawys close the range, regardless of their dates relative to each other. This means that a user can hypothetically click the last day of their range _before_ they click the starting day. Any clicks after a range is closed will always initiate a new range selection, in which the same logic applies.

If this is not desirable, you can use your own logic, and instead, pass an array containing two dates to the `value` prop of `ADatePicker`. The 0th index should represent the starting date, and the 1st index should represent the ending date.

> Please note that it is up to you to ensure the dates passed to the `value` prop maintain the `[earlierDate, laterDate]` tuple structure. `ADatePicker` does _not_ automatically sort the dates.
>
> While this may seem cumbersome, it keeps the range logic flexible for a variety of different use-cases, and doesn't lock the range selection logic to one-specific implementation.

<Playground
  code={`function Example() {
  const [range, setRange] = useState([]);
  return (
    <ADatePicker
      value={range}
      onChange={(incomingDate) =>
        setRange(([start, end]) => {
          if ((start && end) || (!start && !end)) {
            // Reset. Since we don't have user's second selection,
            // place it at the beginning to start.
            return [incomingDate];
          } else {
            // Now that we have the two selections, sort them
            return [start, incomingDate].sort(
              (a, b) => Date.parse(a) - Date.parse(b)
            );
          }
        })
      }
    />
  );
}
  `}
/>

#### Limiting Date Range Selection

Sometimes, you might want to restrict the number of days the user can select in the range, like if you are trying to prevent a user from downloading a full months of log data that could potentially overwhelm the server on such a request.

This is possible by dynamically setting the `minDate` and `maxDate` props after the user's first range selection. Since that can be non trival to implement, it's also a configuration option in `useADateRange`:

> As of latest Magna React release, it is not possible to have both static and dynamic date restrictions. For example, you cannot limit the user's date range to be within the month of March **and** also restrict their range selection to 7 days.
>
> If this is something you need, please consider [opening an feature request in GitHub](https://github.com/cisco-sbg-ui/magna-react/issues/new?assignees=&labels=&template=feature_request.md&title=).

<Playground
  code={`function Example{
  const today = new Date();
  const weekFromToday = new Date(today);
  weekFromToday.setDate(today.getDate() + 7);
  const { value, onChange, minDate, maxDate } = useADateRange({
    initialRange: [],
    maxDays: 7 // Will restrict second selection to 6 days before/after the first selection
  });
  return (
    <ADatePicker
      value={value}
      onChange={onChange}
      minDate={minDate}
      maxDate={maxDate}
    />
  );
}
`}
/>

Or, with your own logic:

<Playground
  code={`function Example() {
  const MAX_RANGE = 3;
  // Usually, you'd define reducer outside of the component, but we are limited
  // in the docs example playground.
  const reducer = useCallback(function rangeReducer(state, action) {
    const { type, payload } = action;
    switch (type) {
      case "SET_NEW_RANGE":
        // Initialize the range restriction
        const base = new Date(payload);
        const minDate = new Date(base);
        const maxDate = new Date(base);\n
        minDate.setDate(base.getDate() - (MAX_RANGE - 1));
        maxDate.setDate(base.getDate() + (MAX_RANGE - 1));\n
        return {
          ...state,
          range: [payload],
          minDate,
          maxDate,
        };\n
      case "SET_FINAL_RANGE":
        const sorted = [payload, state.range[0]].sort(
          (a, b) => Date.parse(a) - Date.parse(b)
        );\n
        return {
          ...state,
          range: sorted,
          minDate: null,
          maxDate: null,
        };\n
      default:
        return { ...state };
    }
  }, []);\n
  const [state, dispatch] = useReducer(reducer, {
    range: [],
    minDate: null,
    maxDate: null,
  });\n
  const handleChange = (incomingDate) => {
    const [start, end] = state.range;
    const isPickerEmpty = !start && !end;
    const isPickerFull = start && end;
    const type =
      isPickerEmpty || isPickerFull ? "SET_NEW_RANGE" : "SET_FINAL_RANGE";\n
    dispatch({ type, payload: incomingDate });
  };\n
  return (
    <ADatePicker
      value={state.range}
      onChange={handleChange}
      minDate={state.minDate}
      maxDate={state.maxDate}
    />
  );
}
`}
/>
